How-To Guide

Everything you need to know about using the Collierville Operations Hub

Add to Your Phone (Home Screen App)

You can add the Collierville Operations Hub to your phone's home screen so it opens like a regular app — no app store needed.

iPhone (Safari)
  1. Open the hub URL in Safari (this does NOT work in Chrome on iPhone)
  2. Tap the Share button at the bottom of the screen (the square with an arrow pointing up)
  3. Scroll down and tap "Add to Home Screen"
  4. Name it "Collierville Hub" (or whatever you prefer)
  5. Tap Add in the top right
  6. The app icon will now appear on your home screen — tap it to open the hub full-screen like a real app
Note: You MUST use Safari on iPhone. The "Add to Home Screen" option does not appear in Chrome, Firefox, or other browsers on iOS.
Android (Chrome)
  1. Open the hub URL in Chrome
  2. Tap the three-dot menu (⋮) in the top right corner
  3. Tap "Add to Home screen" (or "Install app" if it appears)
  4. Name it "Collierville Hub"
  5. Tap Add
  6. You may get a popup asking to add — tap "Add" again to confirm
  7. The app icon will now appear on your home screen
Tip: Once added to your home screen, the hub opens in full-screen mode without the browser address bar, so it looks and feels like a native app.

Using the Checklists (Opening, Closing, Weekly)

  1. Click the Opening, Closing, or Weekly Cleaning button at the top to open that checklist.
  2. As each task is completed, check the box next to it. The task will get a strikethrough to show it's done.
  3. Select the name of the person who completed it from the "Done by" dropdown on the right side of each task.
  4. The progress bar at the top shows how many tasks are done out of the total.
  5. Your progress auto-saves in the browser — you can close the page and come back without losing anything.
  6. When the shift is over, click "Submit to Google Sheets" to log the day's completion to the Google Sheet.
  7. Click "Reset" to clear all checkboxes and dropdowns for the next day/shift.
Tip: Always submit to Google Sheets BEFORE resetting, so the day's work is recorded.

Using the Prep Sheet

  1. Click Prep Sheet from the top menu.
  2. The Par column shows how many of each item you should have.
  3. Type the current count into the On Hand column.
  4. The Prep column auto-calculates how many you need to make (Par minus On Hand).
  5. A green 0 means you're at par — no prep needed for that item.
  6. A red number means you need to prep that many.
  7. Click "Submit to Google Sheets" to log the prep data for the day.
Remember: Fingers are prepped as soon as they come off the truck. Shred cabbage when you get down to 3 pans of slaw.

Google Sheets Setup (One-Time)

  1. Create a new Google Sheet and name it "Collierville Operations Log"
  2. Click Extensions in the top menu bar, then click Apps Script
  3. You'll see a code editor. Delete everything in it
  4. Open the file google-apps-script.js from the project folder, copy all the code, and paste it into the Apps Script editor
  5. Look at the toolbar — next to the Run button (the play/triangle icon ▶), there is a dropdown that says a function name. Click that dropdown and select setupSheets
Toolbar looks like:   [ ▶ Run ]   [ Debug ]   [ setupSheets ▼ ]
                                                      ↑ click this dropdown
  1. Click Run (▶). Google will ask you to authorize — click through the prompts to allow it
  2. You should see a popup that says "Setup complete! Tabs created with headers."
  3. Go back to your Google Sheet — you'll now see 4 tabs: Opening, Closing, Weekly, Prep with headers
  4. Back in Apps Script, click Deploy → New Deployment
  5. Click the gear icon next to "Select type" and choose Web app
  6. Set "Execute as" to Me
  7. Set "Who has access" to Anyone
  8. Click Deploy and authorize again if prompted
  9. Copy the Web App URL it gives you
  10. Open index.html in a text editor, find SHEETS_URL, and paste the URL between the quotes
Important: If you ever update the Apps Script code, you must create a New Deployment (not just save) for changes to take effect.

Troubleshooting

  • Checkboxes not saving? — Make sure you're not in a private/incognito browser window. localStorage doesn't persist there.
  • Submit not working? — Make sure the Google Apps Script URL is pasted correctly in index.html and the deployment is set to "Anyone" access.
  • Headers missing in Google Sheet? — Run the setupSheets function again from Apps Script.
  • Data not appearing after submit? — Check that the tab names in Google Sheets match exactly: Opening, Closing, Weekly, Prep (case-sensitive).
